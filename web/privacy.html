Privacy

Local by design.

CtxAnt drives your real Chrome from Telegram, but it runs on your machine. There is no CtxAnt cloud account, no hosted browser session, and no relay server in the middle.

TL;DR. The extension talks to the app over ws://127.0.0.1. The only time task data leaves your machine is when CtxAnt sends context to the model provider you chose with your own API key.

What the extension can access

When you explicitly ask an agent to operate on a page, the extension can inspect the active tab, take screenshots, click, type, scroll, and navigate. It does not passively observe browsing behavior outside that command flow.

Where data goes

  • Localhost only: browser control happens over the local WebSocket and pairing HTTP endpoints on your machine.
  • Your model provider: task context goes directly from your machine to xAI or Anthropic using your own API key.
  • Telegram: your bot messages move through Telegram like any other Telegram chat.
  • Nowhere else: CtxAnt does not run analytics, user accounts, or a hosted browsing backend.

What is stored locally

  • ~/Library/Application Support/ctxant/.env for your Telegram token and model key.
  • The local SQLite database for agent memory, schedules, usage, and browser pairing state.
  • Per-agent memory that you explicitly provide during setup flows.

Extension permissions

  • tabs, activeTab, scripting to drive the pages you told the agent to work on.
  • storage and alarms to keep the extension paired and connected.
  • <all_urls> because the whole point is to let your agent act on the sites you actually use.
  • http://127.0.0.1/* and http://localhost/* for the local pairing flow.
